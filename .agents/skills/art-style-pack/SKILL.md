# Art Style Pack

## When to use
Create a lightweight art direction pack with pillars, references, asset priorities, readability, and pipeline constraints. Use early in visual development or when coherence drifts.

## Inputs to gather
- visual fantasy
- camera distance
- platform/perf limits
- team size

## Recommended roles
- `art_director`
- `concept_artist`
- `technical_artist`

## Primary docs / outputs
- `studio/docs/templates/art-direction-lite.md`
- `studio/docs/templates/content-pipeline.md`

## Workflow
1. Inspect the current repo/docs state first and cite concrete evidence.
2. Choose the smallest useful output that moves the project forward.
3. Translate visual/audio direction into concrete constraints for production and implementation.
4. Update or create durable docs when the result should persist.
5. Recommend the next best role or skill if more work remains.

## Category rules
- Optimize for clarity and production sustainability, not only taste.
- Note runtime, content-pipeline, and performance implications of artistic choices.
- Keep deliverables concise enough for downstream implementation.

## Deliverables
- art direction pack
- asset priority list
- readability rules
- pipeline cautions

## Validation bar
- style/readability rules captured
- pipeline implications noted
- asset or implementation next step clear
