# Documentation Instructions

These instructions apply under `studio/docs/`.

## Goals
- Keep docs durable, concise, and execution-friendly.
- Update the nearest existing source of truth before creating a new doc.
- Record meaningful decisions, risks, and validation notes that should survive chat history.

## Writing rules
- Use concrete nouns: files, systems, scenes, milestones, owners, dates.
- Prefer checklists, tables, and short sections over prose sprawl.
- Link related docs by filename when the relationship matters.
