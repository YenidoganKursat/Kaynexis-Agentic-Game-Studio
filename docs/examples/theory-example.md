# Theory Example

## Scope

A tactical RPG tutorial and first encounter pass that needs a clearer theory stack before the gameplay slice grows.

## Baseline

- The core loop exists, but the experience is too dense in the first five minutes.
- The player can execute actions, but the reason the actions matter is not yet fully legible.
- The current design needs a tighter explanation of challenge, mastery, and learning order.

## Decision order

1. Player outcome
2. Dominant tension
3. Lens stack
4. Evidence source
5. Narrow proof path

## Example theory pack

| Part | Example |
|---|---|
| Player outcome | The player should feel smart, in control, and rewarded for planning ahead. |
| Lens stack | MDA, GameFlow, SDT/PENS, affordance, and cognitive load |
| Evidence | The tutorial requires too many labels before the first meaningful decision |
| Design change | Reduce the first decision set, add one visible reward loop, and separate the explanation of terrain from the explanation of combat |
| Validation | Run one short battle, one tutorial pass, and one small playtest before expanding the content |

## Good agent prompts

- "Build a theory stack for the first tactical RPG encounter and keep MDA, GameFlow, and SDT separate."
- "Review the tutorial for cognitive load and affordance before adding more content."
- "Turn this combat slice into a theory pack the next agent can reopen later."

## Validation

- The theory lens list is explicit.
- The evidence matches the claim.
- The first proof path is narrow enough to run quickly.
