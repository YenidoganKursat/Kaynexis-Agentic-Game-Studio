# Foundations

Use this folder when a task needs deeper theoretical grounding than engine docs or genre notes alone can provide.

## Fast picks

| If the task is about... | Start with |
| --- | --- |
| the fastest safe start, short bundle, or urgent narrow task | `agent-speedpack.md`, `docs/reference/agent-speedpack.md`, `docs/examples/agent-speedpack-example.md` |
| controller policy, hierarchy, transcripts, or validation matrices | `agent-system.md`, `mastermind.md`, `agent-portfolio.md`, `agent-hierarchy.md`, `agent-validation-matrix.md`, `prompt-journal.md`, `agent-transcript.md` |
| execution packets or pre-flight work contracts | `agent-execution.md` and `docs/examples/agent-execution-example.md` |
| the repo's full capability surface or a newcomer intro | `docs/reference/capabilities.md`, `capabilities.md`, and `docs/examples/capabilities-example.md` |
| recurring engine bugs, common errors, or troubleshooting surfaces | `engine-bugs.md` and `docs/examples/engine-bugs-example.md` |
| engine choice, fit, or engine readiness | `engine-fit.md` and `engine-eval.md` |
| performance, GPU, or scale decisions | `perf-algorithms.md`, `gpu.md`, `benchmarks.md` |
| custom packs, house rules, or extension packs | `custom-packs.md`, `custom-architecture.md`, and `extensions.md` |
| console readiness, submission gates, or cert evidence | `docs/reference/console-guide.md`, `docs/examples/console-example.md`, and `docs/research/game-development/production/console.md` |
| theory stacks, MDA, GameFlow, or SDT/PENS | `theory.md` |
